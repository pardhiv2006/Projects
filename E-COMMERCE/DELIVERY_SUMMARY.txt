## ğŸ”’ How It Works

### Order Creation
1. User clicks "PLACE ORDER" in Cart
2. Frontend prepares order data
3. Calls POST `/api/orders`
4. Backend validates and saves to MongoDB
5. Returns order ID and details
6. Frontend stores order locally
7. Redirects to Orders page

### Order Cancellation
1. User clicks "Cancel Order"
2. Frontend calls PATCH `/api/orders/:id`
3. Backend updates status to "Cancelled"
4. Frontend updates local state
5. UI reflects immediately

### Order Delivery
1. User clicks "Mark as Received"
2. Frontend calls PATCH `/api/orders/:id`
3. Backend updates status to "Delivered"
4. Frontend updates local state
5. UI shows delivery confirmation
